Fetch Installation Access Token (IAT): Use the generated JWT to make a POST request to [https://api.github.com/app/installations/](https://api.github.com/app/installations/){installation_id}/access_tokens using RestClient.

X-GitHub-Api-Version: 2022-11-28

Step 3: Implement Core Controllers & Services
Implement the following REST Controllers and backing Services for the dashboard frontend to consume. Map standard JSON payloads to Java Records.

Fetch the repository's public key from GET /repos/{owner}/{repo}/actions/secrets/public-key.

Use lazy-sodium-java (Libsodium wrapper) to encrypt the plain text secret using crypto_box_seal with the fetched public key.

PUT /repos/{owner}/{repo}/actions/secrets/{secret_name} -> Upload the Base64-encoded encrypted secret.

4. PullRequestController

GET /api/repos/{repo}/pulls -> List open PRs.

PUT /api/repos/{repo}/pulls/{pull_number}/merge -> Execute a merge (allow standard, squash, or rebase).
